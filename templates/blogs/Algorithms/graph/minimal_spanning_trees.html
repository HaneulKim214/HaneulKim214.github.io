What is Minimal Spanning Tree?

Just by the name Minimal Spanning Tree it explains everything. We've already learned that tree is an acyclic undirected graph if you are familiar with graph terminologies I recommend you to check out graph introduction.

Spanning Tree of a graph G is a tree that spans G => include every vertex in G and is a subgraph of G. Spanning tree of a graph must include all vertices but not edges, it can contain all edges or subset of edges.

Minimal Spanning Tree is a spanning tree that have lowest cost => sum of edges' weight are the lowest among other spanning trees.

There are multiple applications of MST in network design such as telephone line, electrical cables, TV cables, computer and more.
Say that there are five cities that your company provides electricity to, from your electrical power plant there are multiple cables that are connected to each of five cities. Now company need to maximize profit therefore will try to find electricity cables that reach all five cities with lowest costs. Another use case can be uber-eats where you want to deliver many houses(verticies) in shortes amount of time which can be done by taking shortest paths from one house to another.

We know what Minimal Spanning Trees are but how to efficiently find them?

Two famous algorithm for finding Minimal Spanning Tree are Kruskal and Prim's algorithm which follow greedy approach.

첫번째 답

다른 답들도 몇가지 봤는데 저는 위에 답이 가장 이해가 잘 됬기 때문에 사용 하였습니다. 이답 저답 다 한번 봐보시길 추천!

실제로 손으로 풀어보신다면 이게 바로 Breadth First Search 를 사용 하고 있다는것을 알수 있을것입니다. numberlist 에 있는 숫자들을 possible_targets 에 넣어주는것인데 각 숫자마다 +- 두가지가 될수 있기때문에 2^n 개의 가능성이 있습니다.

예시: numberlist = [1,2,3], target = 6

  1. +1, -1 두개의 경우의 수가 있습니다. 현재 possible_targets 에 있는 숫자는 0 이기 때문에 lst = [(0+1), (0-1)] = [1,-1] 이 되고 이 lst 가 possible_targets 으로 바뀝니다.
  2. 이제 possible_targets = [1, -1]. 여기서 각 숫자마다 그다음 numberlist 에 있는 2 를 +2, -2 해줍니다.
    [(1 + 2), (1 - 2),
    (-1 + 2), (-1 - 2)]
    이렇게 1에게 +2, -2 두개의 경우의 수가 생기고, -1 에게도 두개의 경우의 수가 생깁니다.
  3. 마지막 숫자를 possible_targets 에 있는 숫자에 +- 한번씩 해준다면 numberlist 가 가질수 있는 모든 +- 경우의수의 답이 나옵니다. 여기서 이제 target 이 몇개나 있는지 possible_targets.count(target) 해주면 끝.
